UMBC DoorDash

Campus Food Delivery Platform

View Project on GitHub

1. Introduction & Motivation

UMBC DoorDash is a full-scale, production-ready campus food delivery platform designed for the University of Maryland, Baltimore County. The system addresses a clear pain point: the lack of a centralized delivery solution on campus. Students previously faced long walks or scattered third-party options that do not integrate with dining halls or campus restaurants well.

UMBC DoorDash provides a seamless, unified experience for students, dining staff, and on-campus drivers, tailored to campus buildings, dining schedules, and spatial constraints.

2. System Architecture

The platform is built on a distributed microservices architecture, decoupling core services to maximize scalability, maintainability, and fault tolerance.

3. Core Features

Distributed Microservice Platform

  • Authentication service with role-based access (student, driver, restaurant admin).
  • Restaurant & Menu Service.
  • Order Service.
  • Driver Coordination Service.
  • Real-Time WebSocket Service with Redis pub/sub for live event streaming.

Real-Time Driver Tracking

  • Drivers send GPS updates every 1â€“2 seconds.
  • Students and restaurants monitor deliveries live.
  • Reduces uncertainty and improves delivery accuracy.

Dynamic Route-Assignment Algorithm

  • Assigns drivers based on location, current workload, restaurant prep time, and dorm-specific constraints.
  • Ensures low-latency dispatching and balanced driver workload.

Restaurant Admin Dashboard

  • React-based interface for real-time menu management, order monitoring, and operational oversight.
  • Track peak order times and adjust open/closed hours dynamically.

Accelerated Backend Infrastructure

  • MongoDB stores all critical data: accounts, user profiles, orders, menus, and location information.
  • Redis used for caching and event streaming.
  • Dockerized microservices behind Nginx load balancer.
  • JWT-secured authentication with hashed credentials.
  • Stripe integration enables seamless in-app payments for students and campus dining, fully integrated with the backend for transaction verification and user account balance updates.

4. Performance & Scalability

  • Load-tested to simulate campus-scale usage: 400+ concurrent users.
  • Sub-200ms API latency under peak load.
  • 1,500+ orders processed during full stress testing.
  • WebSocket propagation delay: < 80ms.
  • Scales easily across multiple dining halls and driver fleets.

6. University Impact

UMBC DoorDash enhances campus life by:

  • Providing a centralized, fast, and reliable food delivery solution.
  • Offering live dashboards for dining staff and campus vendors.
  • Supporting student drivers in earning income while navigating campus efficiently.
  • Enabling event catering, student organization orders, and late-night meal delivery.
  • Seamlessly handling payments and account management through integrated Stripe + MongoDB backend.

7. Future Roadmap

Planned enhancements to refine the platform include:

  • Predictive ML-based order batching and demand forecasting.
  • Heatmap-style peak-time visualization.
  • Push notifications for order updates.
  • Smart delivery zones based on peak demand.
  • Meal plan integration with campus ID payment.
  • Intelligent driver scheduling based on historical workload patterns.
