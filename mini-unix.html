Building a Mini Unix

Custom Linux Kernel and C Shell from Scratch

View Project on GitHub

I engineered a full Linux development environment and custom kernel from scratch, then built a minimal but fully functional Unix-style shell with support for process creation, argument parsing, system execution, and /proc filesystem exploration.

1. Virtual Linux Environment Setup

Built an isolated Linux development environment using VirtualBox (Windows). Installed and configured Debian 11.5.0, ensuring the VM was optimized for resource usage and reproducibility. Established a structured Git workflow to version-control all system-level modifications and track implementation progress.

2. Compiling and Customizing the Linux Kernel

Downloaded and compiled the Linux Kernel (v5.15.67) manually from source. Customized the kernel configuration, regenerated .config, and optimized build settings. Added a custom kernel version string to identify and verify my own compiled kernel on boot. Installed the new kernel image, updated GRUB, and successfully booted into the custom environment.

3. Implementing a New Linux System Call

Extended the kernel syscall table to add a custom system call that prints a message directly from kernel space to the userâ€™s screen. Modified syscall headers, implemented the kernel-mode C function, and rebuilt + reinstalled the kernel. Validated the system call by invoking it from a user-space C program via syscall().

4. Designing and Building a C-Based Shell

Built a minimal, robust shell similar to early Unix shells, with the following components:

Input Parsing & Execution

  • Implemented a full REPL loop with arbitrary-length input using dynamic memory (e.g., getline()).
  • Engineered a tokenizer + parser that handles multiple whitespace delimiters, quoted strings, and escaped characters (\n, \t, \", etc.).
  • Created argument vectors compatible with all exec() variants.
  • Used fork() and execvp() to spawn processes, and waitpid() for synchronization.
  • Supports execution of system programs such as ls, cat, cp, mkdir, rm, echo, and any executable in $PATH.

Exit Command & Error Handling

  • Implemented a built-in exit command with optional integer return codes.
  • Added robust validation for invalid or malformed exit arguments.
  • Ensured complete memory cleanup and child-process termination on exit.

Memory Safety

  • Designed the shell with strict memory-management discipline.
  • Validated correctness and leak-freedom using valgrind --leak-check=full.

5. Integrating /proc Filesystem Support

Added a custom shell feature to let users inspect Linux kernel and process information. My shell recognizes input beginning with /proc/... and:

  • Opens the corresponding file directly using fopen().
  • Reads and prints the contents to stdout (no cat, no external programs).
  • Supports key system files: /proc/cpuinfo, /proc/loadavg, /proc/filesystems, /proc/mounts, /proc/<pid>/status, /proc/<pid>/environ, /proc/<pid>/sched.

This required implementing safe file I/O layers, error-handling routines, and careful parsing of /proc entries.

6. Tools, Workflow, and Developer Practices

Maintained a detailed engineering worklog documenting architecture decisions, debugging steps, and kernel-level changes. Followed a disciplined Git workflow with clear commit histories, state-safe checkpoints, and continuous backups (GitHub + VM snapshots). Designed the entire system to be reproducible, easily recompiled, and compatible with future kernel modifications.
