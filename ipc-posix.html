High-Performance IPC with POSIX Threads

Multi-threaded Pipeline Simulation

Project Overview

Built a multi-threaded pipeline to simulate an operating system’s memory access management. The project implements interprocess communication (IPC) using POSIX threads (pthread.h) and message queues (mqueue.h) to pass memory requests through a staged pipeline. Each stage of the pipeline corresponds to a thread handling a specific task: reading, validating, and processing memory access requests.

Key Objectives

  • Simulate memory request handling in an OS by mapping logical addresses to physical addresses.
  • Implement thread-safe message passing between pipeline stages using mailboxes (message queues).
  • Validate and log memory requests with timestamped output, demonstrating real-world concurrency patterns.
  • Ensure high-priority requests are handled first using message queue prioritization.

Technical Implementation

Stage 1 – Producer Thread (stage1_producer)

  • Reads logical memory access requests from a text file. Each line contains a request ID, access type (read/write), memory address, and priority.
  • Creates a structured message for each request and sends it to the first mailbox queue using mq_send.
  • Integrated randomized delays (usleep) to simulate asynchronous arrival of requests.
  • Used printf to verify real-time thread operation.

Stage 2 – Validator Thread (stage2_con_prod)

  • Receives messages from Stage 1 using mq_receive.
  • Validates the request type and translates logical addresses to physical addresses by mapping (logical address – 500).
  • Writes validated requests to stage2.log with timestamps using time() and ctime().
  • Sends validated requests to the second mailbox queue, ensuring messages are sent with the correct priority.
  • Simulated random processing delays to mimic real OS behavior.

Stage 3 – Consumer Thread (stage3_consumer)

  • Retrieves messages from Stage 2 mailbox, respecting priority order.
  • Converts numeric access type back to human-readable format (“READ” / “WRITE”).
  • Logs all requests to stage3.log with timestamps and prints output to the screen for monitoring.
  • Simulated longer processing delays to reflect downstream workload.

Tools and Libraries

  • C / GCC: Compiled with gcc -pthread -lrt for POSIX threading and real-time IPC support.
  • POSIX Threads (pthread.h): Managed concurrent execution of producer, validator, and consumer threads.
  • Message Queues (mqueue.h): Enabled asynchronous communication and prioritization of memory requests.
  • File I/O & Logging: Used structured logging with timestamps to monitor thread execution and request handling.

Challenges & Solutions

  • Concurrency & Synchronization: Ensured thread-safe access to mailboxes while preserving message priority.
  • Randomized Processing Delays: Used usleep and a custom bsort helper to simulate real-world pipeline timing.
  • Message Structuring: Designed message structures (msg_stage1_t, msg_stage2_t) to efficiently pass data between threads.
  • Memory Safety: Ran valgrind to verify no memory leaks and proper resource cleanup (mq_close and mq_unlink).

Project Outcome

  • Successfully implemented a fully functional, multi-threaded IPC pipeline.
  • Demonstrated ability to design and manage concurrent systems, handle message prioritization, and log structured output.
  • Gained practical experience with low-level OS concepts, interprocess communication, and real-time thread management.
